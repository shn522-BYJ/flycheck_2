# 第一、二阶段算法校正说明

## 已修改

1. **坐标源**
   - 新增并保留 `rtk_lat/rtk_lng`、`f_lat/f_lng`、`gps_lat/gps_lng`。
   - 自动模式优先使用 RTK；融合坐标和 GPS 作为辅助/备份。
   - 不再把较高更新频率等同于更高绝对精度，真实精度仍需外部测量验证。

2. **高频时间轴**
   - 使用 `elapsed_sec` 或 `time` 重建逐行时间。
   - `mission_time_stamp` 仅用于秒级批次对齐，不用于去重。

3. **连续喷洒事件**
   - 以泵开关为主证据切分事件。
   - 泵关闭、长时间断点和明显坐标跳变均会切断轨迹，避免虚假连线。
   - 流量阈值只有在完成标定并由用户传入后才参与判断，代码不自设合格阈值。

4. **标称几何覆盖**
   - `span`/`sprinkle_width` 仅作为任务预设喷幅。
   - 对每个连续喷洒事件分别生成喷洒足迹后求并集。
   - 输出名称改为“标称几何覆盖”，不再称“实际覆盖”。

5. **田块内疑似几何缺口**
   - 支持 GeoJSON/JSON/KML 田块边界及独立排除区。
   - 采用 `目标施药区 - 标称覆盖区` 的多边形差集计算缺口面积和位置。
   - 未提供田块边界时，不再输出“未发现漏喷”。
   - 停用“最近平行线+最小距离”作为主漏喷算法。

6. **药液总量一致性**
   - 新增 `material_balance.py`。
   - 交叉报告流量积分、重量变化、飞控面积增量、亩用量和标称覆盖面积。
   - 重量是质量；未提供药液密度时不换算成体积。
   - 不设置未经实验标定的“合格/不合格”阈值。
   - 明确：总量一致不能证明局部无漏喷。

7. **已修正的其他问题**
   - 作业阶段不再要求高度传感器有效后才承认泵开启喷洒。
   - 喷洒时长改为逐行累计泵开启时间，排除中间关泵间隔。
   - `hAcc` 只报告原始值，不擅自换算成厘米或据此保证定位精度。

## 验证情况

- 3 个合成测试通过：
  1. 泵关闭区不会被跨段连接；
  2. 田块差集面积与理论面积一致；
  3. 流量—重量—面积总量计算逻辑可复现。
- 对提供的 50 个原始 CSV 做了烟雾测试：50/50 均完成解析和标称覆盖计算，无运行异常。

## 尚不能由代码证明

- 任务预设喷幅等于实际有效喷幅；
- 实际雾滴沉积边界；
- 风致漂移位置；
- 冠层沉积与防治效果；
- 厘米级实际漏喷定位；
- 法定意义上的“实际漏喷认证”。

上述内容仍需要外部定位验证、田块边界、气象数据、水敏纸/示踪剂沉积真值、明确阈值及正式检验程序。
