# FlyCheck · 植保无人机飞行日志作业质量筛查与合规追溯工具

基于植保无人机飞行日志（拓攻等）的作业质量**快速筛查**与法规**合规追溯**系统。

> **诚实边界**：本工具仅对**飞行日志中可客观提取**的项做筛查，
> 不替代田间雾滴密度/沉积、药效等检测（须水敏纸/量筒/荧光法实测），
> 也不覆盖实名登记、操作资质、空域审批、农药合法性等**须线下核验**的监管要件。
> 系统只回答"日志能证明什么"，不假装能做日志做不到的事。

## 用户需提供的参数

飞控日志里没有、只有用户知道的，仅一项：

- **作物类型**（小麦 / 大田 / 果树）——用于作业速度/高度的合理性参考。
  飞控日志中无作物信息（厂商原话："用户不会、也没必要告知我们"）。

**选填**：作业风速、敏感区经纬度（用于飘移风险评估）。
> 敏感区经纬度门槛较高，**建议由飞手/技术员填写**；不填则跳过飘移评估、不影响其他结果。
> 手机查坐标：高德/百度 App 默认不显示经纬度——可用微信搜「经纬度查询」小程序、iPhone 自带「指南针」App，或站到敏感区旁读手机定位。
**收费面积**（选填）：农户可在"面积核对"处填飞手报的亩数，当场比对。

> 「设定亩用量」「作业幅宽」等已可从飞控 `dosage`、`span` 字段**自动读取**，无需手填。
> （旧版的"规划作业面积"输入已移除——它仅用于覆盖率，而该值人填、口径不一致，
> 现改为：上传田块边界后计算田块内疑似几何覆盖缺口。）

## 主要功能

### 农户视图（默认首页，大白话）
- **药量**：实际亩用量 vs 飞手设定，判断有没有按目标打；亩用量偏高时提示（结合作业速度）。
- **标称覆盖与疑似缺口**：按连续喷洒轨迹和任务预设喷幅重建标称覆盖；上传田块边界后计算几何差集。结果需现场核实，不等于实际沉积漏喷。
- **面积与总量核对**：比较标称轨迹覆盖、飞控累计面积增量、流量积分和药液重量变化；不把总量一致解释为局部无漏喷。
- **飘移风险**：标注敏感区后评估药液是否可能飘到不该打的地方。

### 合规检查（10 项，分三维度）
- **法规红线**（超出即涉嫌违规）：飞行速度 ≤ 50 km/h、真高 ≤ 30 m、半径 ≤ 2000 m
- **技术参考项**（GB/T 43071 §6.2.2 产品级台架指标，**不参与违规判定**）：高度稳定性、速度稳定性
- **作业条件 / 质量**：喷雾量达标性（亩用量偏差 ±5%，GB/T 43071 §6.2.8）、
  作业速度合理性、作业高度合理性、作业安全距离、作业风速（气象记录项）

### 标称几何覆盖
- 优先使用 `rtk_lat/rtk_lng`，保留 `f_lat/f_lng` 与 `gps_lat/gps_lng` 作为辅助/备份；真实精度需外部测量验证。
- 按泵开关切分连续喷洒事件，避免跨关泵段错误连线。
- 按任务预设喷幅重建标称覆盖。
- 提供 GeoJSON/KML 田块边界后，计算田块内疑似几何缺口面积；未提供边界时不输出“未发现缺口”。

### 药液总量一致性
- 比较流量计积分、重量变化、飞控面积增量、亩用量和标称覆盖面积。
- 未提供药液密度时，重量不换算为体积。
- 不设未经实验验证的合格阈值。

### 设备状态（Battery Doctor·扩展项，非合规判定）
- 电池电量、电池电压、电池温度、电机负载（自动识别 4~8 旋翼）、**喷洒泵/喷头健康**（双泵流量均衡诊断）。

### PDF 报告
- 一键生成可下载的作业质量与合规追溯凭证，含**监管边界声明**（如实交代不覆盖的项）。

## 依据的标准/法规

- 《无人驾驶航空器飞行管理暂行条例》（限高/速/距的农用类别界限）
- GB/T 43071—2023《植保无人飞机》（§5.4 限值、§3.9 作业幅宽、§6.2.8 喷雾量偏差等）
- NY/T 4258/4259/4260（安全施药、安全距离、小麦作业规程）

> 依据均已核对标准原文。凡"产品级台架指标"（如 §6.2.2、§6.2.8）均标注为
> "参照适用、非田间作业合格标准"；自设的工程阈值（如面积/漏喷）均如实标注"无国标合格线"。

## 数据源说明

- 输入：拓攻（TopXGun）植保无人机导出的**原始飞行日志 CSV**（BOM + 表头在第 2 行）。
- 部分字段有已知局限并已在系统中如实处理，例如：
  - 飞控面积字段可能重置，系统采用喷洒期间累计正增量作为参考，并与标称轨迹覆盖分别报告；
  - `liquid_left` 为质量记录且受晃动影响；系统采用首尾窗口中位数，只作总量交叉校验；
  - `spray_real_dosage`（实测亩用量，L/亩）在慢速大流量架次会偏高，属真实值而非异常。


## v3.0 方法边界

- `span` / `sprinkle_width` 是同一任务预设喷幅，不是实时有效喷幅。
- 当前代码能计算标称几何覆盖和田块内疑似几何缺口，不能仅凭日志证明实际雾滴沉积。
- 风速不在飞控日志中；无现场风速/风向时，漂移相关项目只能标记为无法核验。
- “实际漏喷认证”仍需田间沉积真值、明确阈值、独立验证和正式检验程序。
